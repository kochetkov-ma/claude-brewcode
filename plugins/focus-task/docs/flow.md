# Диаграммы потоков focus-task

## Содержание

| # | Диаграмма | Описание |
|---|-----------|----------|
| a | Создание спецификации | `/focus-task:spec` — исследование и SPEC.md |
| b | Создание плана | `/focus-task:plan` — SPEC.md в PLAN.md |
| c | Выполнение задачи | `/focus-task:start` — полный цикл с хуками |
| d | Жизненный цикл lock-файла | Создание, привязка, проверки, удаление |
| e | Handoff при компактификации | Сохранение контекста при auto-compact |
| f | Жизненный цикл задачи | Конечный автомат состояний |

### c.2) Взаимодействие хуков при выполнении

```
  Хук                    Событие                   Действие
  ════════════════════    ══════════════════════     ══════════════════════════

  session-start.mjs      SessionStart               Логирование session_id,
                                                     привязка сессии к задаче,
                                                     линковка LATEST.md (clear)

  pre-task.mjs           PreToolUse:Task             1. grepai напоминание
                         (ПЕРЕД каждым агентом)       2. Инъекция ## K знаний
                                                     3. Ролевые ограничения

  post-task.mjs          PostToolUse:Task            Для координатора:
                         (ПОСЛЕ каждого агента)        привязка session -> lock
                                                     Для рабочих агентов:
                                                       напоминание 2-step протокол

  pre-compact.mjs        PreCompact                  1. Проверка lock
                         (при auto-compact)           2. Валидация артефактов
                                                     3. Компакт KNOWLEDGE
                                                     4. Запись handoff-записи
                                                     5. Статус -> "handoff"
                                                     6. systemMessage с контекстом

  stop.mjs               Stop                        Задача не завершена?
                         (при попытке выхода)           -> БЛОКИРОВАТЬ выход
                                                     Задача завершена?
                                                       -> Удалить .lock
                                                     Stale lock (>24ч)?
                                                       -> Автоочистка
```

## d) Жизненный цикл lock-файла

```

  ┌──────────────────────────────────────────────────────────────────────┐
  │                                                                      │
  │   /focus-task:start                                                  │

  │        ▼                                                             │
  │   ┌─────────────────┐                                               │
  │   │ 1. СОЗДАНИЕ     │  ft-coordinator (mode: initialize)            │
  │   │                 │                                               │
  │   │ {               │                                               │
  │   │  "task_path":.. │  session_id ЕЩЕ НЕ ПРИВЯЗАН                  │
  │   │  "started_at":..│                                               │
  │   │ }               │                                               │
  │   └────────┬────────┘                                               │
  │            │                                                         │
  │            ▼                                                         │
  │   ┌─────────────────┐                                               │
  │   │ 2. ПРИВЯЗКА     │  post-task.mjs (после координатора)           │
  │   │    СЕССИИ       │                                               │
  │   │                 │                                               │
  │   │ {               │                                               │
  │   │  "task_path":.. │  + session_id ПРИВЯЗАН                        │
  │   │  "started_at":..│                                               │
  │   │  "session_id":..│  Теперь lock принадлежит конкретной сессии    │
  │   │ }               │                                               │
```
