# Brewpage

Brewpage publishes text, markdown, JSON, or files to **[brewpage.app](https://brewpage.app)**
and returns a public URL instantly. No sign-up, no configuration — just content in, link out.

<Callout type="tip" title="brewpage.app">
  Free instant hosting for HTML pages, JSON documents, files, and KV stores.
  Visit <a href="https://brewpage.app" target="_blank" rel="noopener noreferrer">brewpage.app</a> to browse the public gallery.
</Callout>

```bash
/brewdoc:publish "Hello, world!"
/brewdoc:publish report.md
/brewdoc:publish '{"status": "ok", "count": 42}'
/brewdoc:publish screenshot.png --ttl 1
```

## How it works

<Steps>
  <li class="step step-primary">
    <strong>Receive content</strong>
    <p>The skill receives user input: inline text, a file path, or a JSON literal.</p>
  </li>
  <li class="step step-primary">
    <strong>Detect content type</strong>
    <p>Content is classified as HTML (text/markdown), JSON (object or array), or File (local path). The type determines which API endpoint will be used.</p>
  </li>
  <li class="step step-primary">
    <strong>Ask namespace</strong>
    <p>An interactive prompt asks for a namespace — a short slug that becomes part of the URL (e.g. <code>brewpage.app/my-ns/abc123</code>).</p>
  </li>
  <li class="step step-primary">
    <strong>Ask password</strong>
    <p>Optionally set a password. Visitors will need to enter it before viewing the page. Leave empty for public access.</p>
  </li>
  <li class="step step-primary">
    <strong>Render content</strong>
    <p>For text/markdown, the content is converted to HTML. JSON is formatted for display. Files are prepared for upload as-is.</p>
  </li>
  <li class="step step-primary">
    <strong>Call API</strong>
    <p>The skill sends a request to the appropriate brewpage.app endpoint: <code>POST /api/html</code>, <code>POST /api/json</code>, or <code>POST /api/files</code>.</p>
  </li>
  <li class="step step-primary">
    <strong>Return URL</strong>
    <p>The API responds with a public URL. The skill displays it immediately.</p>
  </li>
  <li class="step step-primary">
    <strong>Save owner token</strong>
    <p>The owner token, namespace, page ID, and URL are appended to <code>.claude/brewpage-history.md</code> for future reference and deletion.</p>
  </li>
</Steps>

## Content type detection

| Input | Detected type | Endpoint |
|-------|---------------|----------|
| Plain text or markdown string | HTML | `POST /api/html` |
| Valid JSON object or array | JSON | `POST /api/json` |
| Path to an existing local file | File | `POST /api/files` |

## Namespace selection

The namespace is a short identifier that forms part of the published URL.
It is prompted interactively during each publish.

- Must contain only lowercase letters, numbers, and hyphens
- Becomes the first path segment: `brewpage.app/{namespace}/{id}`
- Reusing a namespace groups pages under the same prefix

## Password protection

When prompted, you can set a password to restrict access. Visitors must enter the
correct password before the page content is displayed. Leave the prompt empty for
unrestricted public access.

## Owner token and history

Every successful publish appends a row to `.claude/brewpage-history.md`:

```markdown
| Namespace | ID | URL | Token | Created |
|-----------|-----|-----|-------|---------|
| my-ns | abc123 | https://brewpage.app/my-ns/abc123 | tok_... | 2026-03-31 |
```

To delete a published page, use the saved owner token:

```bash
curl -X DELETE "https://brewpage.app/api/{ns}/{id}" -H "X-Owner-Token: {token}"
```

<Callout type="tip" title="TTL">
  Default time-to-live is 5 days. Override with <code>--ttl N</code> where N is the number of days.
  Example: <code>/brewdoc:publish report.md --ttl 30</code> keeps the page for 30 days.
</Callout>
