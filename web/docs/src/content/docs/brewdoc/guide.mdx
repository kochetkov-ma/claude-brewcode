# Guide

Guide is an interactive teaching skill that walks you through every feature of the
brewcode/brewdoc/brewtools plugin suite. It tracks your progress, supports multiple
languages, and adapts to your level.

```bash
/brewdoc:guide
/brewdoc:guide overview
/brewdoc:guide killer-flow
```

<Callout type="tip" title="Haiku-powered">
  Guide runs on the haiku model for fast, lightweight responses.
  It is read-only and never modifies your project files.
</Callout>

## How It Works

The guide follows a 5-phase flow:

<Steps>
  <li>
    <div>
      <strong>Validate environment</strong>
      <p>Check docs site, GitHub release, installed plugin versions. Show health table.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Language and progress</strong>
      <p>Select language (EN/RU/PT), load progress from previous sessions.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Menu or direct topic</strong>
      <p>Show interactive menu or jump to a topic by name.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Deliver topic</strong>
      <p>Present 3-4 sections with pauses for questions, examples, and deep dives.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Update progress</strong>
      <p>Mark topic complete, recommend next topic.</p>
    </div>
  </li>
</Steps>

Between sections, you can choose:
- **Continue** to the next section
- **Show me an example** with project context
- **Go deeper** for more detail
- **Skip** or **Back to menu**

## Progress Tracking

Progress is saved to `~/.claude/brewdoc/guide-progress.json` and persists across sessions.

```json
{
  "lang": "en",
  "completed": ["overview", "installation"],
  "last_topic": "killer-flow",
  "last_ts": "2026-04-02T10:00:00Z",
  "shown_count": { "overview": 2, "installation": 1 }
}
```

Returning users see their completion status and a recommendation for the next topic.

Two helper scripts manage the guide:

| Script | Purpose |
|--------|---------|
| `validate.sh` | Check docs site, GitHub release, installed versions, auto-update status |
| `progress.sh` | CRUD for progress JSON: `read`, `complete`, `lang`, `reset`, `status` |

## Environment Health Check

On every launch, the guide checks your environment and shows a health table:

```
Component     Status      Details
Docs Site     UP          200
Latest        v3.4.20     GitHub
brewcode      3.4.20      current
brewdoc       3.4.20      current
brewtools     3.4.20      current
Auto-update   ON
```

<Callout type="note" title="Direct topic access">
  Jump straight to any topic: <code>/brewdoc:guide killer-flow</code>.
  Partial matching works too: <code>/brewdoc:guide agent</code> resolves to the agents catalog.
</Callout>
