# glm-openrouter-specialist

<Callout type="caution">
**OpenRouter adds a routing layer between Claude Code and the GLM backends.** Model IDs, auth headers, and provider fallback chains differ from the direct Z.ai API. Sending a direct-API payload to OpenRouter — or vice versa — returns a 400 or silent truncation. This agent handles the translation.
</Callout>

<Callout type="tip">
**Auto-delegated by [`/brewui:glm-design-to-code`](/brewui/skills/glm-design-to-code/)** when OpenRouter is selected as provider. You can also invoke it directly for standalone API requests, cost audits, or model selection decisions.
</Callout>

## Quick reference

| Field | Value |
|-------|-------|
| Name | `glm-openrouter-specialist` |
| Model | sonnet |
| Tools | Read, Bash, Glob, Grep, Write, Edit |
| Triggers | "openrouter", "open router", "route glm", "openrouter api", "glm via openrouter", "glm provider", "openrouter pricing" |
| Env required | `OPENROUTER_API_KEY`, `BU_PLUGIN_ROOT` (injected by hook) |

## When to use

- **Design-to-code via OpenRouter** — send a screenshot or image to GLM through the OpenRouter gateway
- **Model selection** — pick the right GLM variant (quality vs. cost vs. free tier) for the task
- **Cost audit** — estimate or review token spend before a batch run
- **Provider routing** — lock to a specific backend (e.g., SiliconFlow) or configure fallback chains
- **Debugging 4xx/5xx** — diagnose auth errors, rate limits, and provider downtime

## Examples

```
"use openrouter to send this screenshot to glm"
```

## Flow

<Steps>
  <li>
    <div>
      <strong>Validate env</strong>
      <p>Checks <code>OPENROUTER_API_KEY</code> is set and valid via <code>/api/v1/auth/key</code>. Reads <code>BU_PLUGIN_ROOT</code> from prompt context (injected by <code>pre-task.mjs</code>). If either is missing — stops with an explicit error before touching any files.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Select model</strong>
      <p>Matches the task to the optimal GLM variant: vision tasks get <code>z-ai/glm-5v-turbo</code> (quality) or <code>z-ai/glm-4.6v</code> (budget); text-only tasks use <code>z-ai/glm-4.5-air:free</code> at zero cost. Model IDs must carry the <code>z-ai/</code> prefix — bare names are rejected by OpenRouter.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Build payload</strong>
      <p>Calls <code>glm-build-request.sh</code> from <code>$BU_PLUGIN_ROOT/skills/glm-design-to-code/scripts/</code> to construct the JSON payload with image (base64-encoded), prompt, model ID, and <code>max_tokens</code>. Optionally adds <code>provider.order</code> and <code>provider.allow_fallbacks</code> for backend routing control.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Send request</strong>
      <p>Runs <code>glm-request.sh payload.json response.json openrouter</code>. The script posts to <code>https://openrouter.ai/api/v1/chat/completions</code> with the required <code>Authorization</code> and <code>Content-Type</code> headers, plus optional <code>HTTP-Referer</code> and <code>X-Title</code> for OpenRouter dashboard identification.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Parse response</strong>
      <p>Checks HTTP status code and <code>finish_reason</code>. Extracts content via <code>jq -r '.choices[0].message.content'</code>. For multi-file design-to-code output, delegates extraction to <code>glm-extract.sh</code> which splits on <code>===FILE:===</code> / <code>===END_FILE===</code> markers.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Report cost</strong>
      <p>Reads <code>.usage</code> from the response — OpenRouter returns <code>prompt_tokens</code>, <code>completion_tokens</code>, and <code>total_cost</code>. Prints a summary table: model, finish reason, token counts, estimated USD cost. Surfaces any 402/429/5xx errors with a concrete remediation step.</p>
    </div>
  </li>
</Steps>

**Provider routing snippet:**

```json
{
  "provider": {
    "order": ["SiliconFlow"],
    "allow_fallbacks": true,
    "require_parameters": true,
    "data_collection": "deny"
  }
}
```

**Error reference:**

| HTTP | Meaning | Fix |
|------|---------|-----|
| 400 | Bad request | Check payload format, verify `z-ai/` model prefix |
| 401 | Auth failed | Verify `OPENROUTER_API_KEY` |
| 402 | Insufficient credits | Top up OpenRouter account |
| 429 | Rate limited | Wait and retry with exponential backoff |
| 502/503 | Provider down | Retry or add `provider.order` fallback |

</details>
