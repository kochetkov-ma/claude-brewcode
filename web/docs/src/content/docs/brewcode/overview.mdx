# Brewcode Overview

<InstallPrompt plugin="brewcode" mode="install" />

Install the entire suite:

<InstallPrompt plugin="all" mode="install" />

Brewcode is a lean **skill + prompt-injection toolkit** for Claude Code. It ships two focused skills --
[spec](/brewcode/skills/spec/) for structured task specifications and [grepai](/brewcode/skills/grepai/)
for semantic code search -- backed by nine developer-facing agents and five lifecycle hooks that wire
everything into the Claude Code session. The name is a nod to brewing: skills (recipes) define what to
brew, agents (brewers) do the work, and hooks (processes) manage the lifecycle.

## Architecture

Brewcode works as a layer on top of Claude Code through the hook system. The plugin has no external runtime --
everything is managed through built-in Claude Code events.

```
                         Brewcode Plugin
  ┌──────────────────────────────────────────────────────────┐
  │                                                          │
  │   Skills (recipes)            Agents (brewers)           │
  │   ┌──────────────┐          ┌──────────────────┐         │
  │   │ spec         │          │ developer, tester │         │
  │   │ grepai       │ ──────>  │ reviewer, architect│         │
  │   │              │          │ creators, bash    │         │
  │   └──────────────┘          └──────────────────┘         │
  │          │                          │                     │
  │          v                          v                     │
  │   Hooks (processes)                                       │
  │   ┌──────────────────────────────┐                       │
  │   │ session-start, forced-eval   │                       │
  │   │ grepai-session, grepai-remind │                       │
  │   │ permission-guard             │                       │
  │   └──────────────────────────────┘                       │
  │                                                          │
  └──────────────────────────────────────────────────────────┘
```

## Plugin structure

```
brewcode/
├── .claude-plugin/
│   └── plugin.json                    # Plugin manifest
├── hooks/
│   ├── hooks.json                     # Binding 5 hooks to events
│   ├── session-start.mjs              # Session initialization
│   ├── grepai-session.mjs             # Auto-start grepai watch
│   ├── grepai-reminder.mjs            # grepai reminder (Bash matcher)
│   ├── forced-eval.mjs                # Skill activation boost
│   ├── permission-guard.sh            # Auto-allow safe .claude/ writes
│   └── lib/
│       └── utils.mjs                  # I/O, configuration, logging
├── agents/                            # 9 agents
│   ├── developer.md                   # Feature implementation (opus)
│   ├── tester.md                      # Testing (sonnet)
│   ├── reviewer.md                    # Code review (opus)
│   ├── architect.md                   # Architecture (opus)
│   ├── skill-creator.md               # Skill creation (opus)
│   ├── agent-creator.md               # Agent creation (opus)
│   ├── hook-creator.md                # Hook creation (opus)
│   ├── bash-expert.md                 # Bash scripts (opus)
│   └── bc-grepai-configurator.md      # grepai configurator (opus)
└── skills/                            # 2 skills
    ├── spec/                          # Specification creation
    └── grepai/                        # Semantic search
```

## Components in detail

<Tabs>
  <TabItem label="Skills">
    Two focused skills. See [Skills](/brewcode/skills/) for details.

    | Skill | Command | Purpose |
    |-------|---------|---------|
    | [spec](/brewcode/skills/spec/) | `/brewcode:spec` | Structured task specifications via parallel research |
    | [grepai](/brewcode/skills/grepai/) | `/brewcode:grepai` | Semantic code search management |
  </TabItem>
  <TabItem label="Agents">
    Nine agents (brewers) with different models and specializations.
    See the [Agents](/brewcode/agents/) section for details.

    | Role | Agents | Model |
    |------|--------|-------|
    | Implementation | developer, tester, architect | opus/sonnet |
    | Review | reviewer | opus |
    | Creation | skill-creator, agent-creator, hook-creator | opus |
    | Tooling | bash-expert, bc-grepai-configurator | opus |
  </TabItem>
  <TabItem label="Hooks">
    Five hooks (processes) manage the session lifecycle through Claude Code events.
    See the [Hooks](/brewcode/hooks/) section for details.

    | Event | Hooks | Purpose |
    |-------|-------|---------|
    | UserPromptSubmit | forced-eval | Skill activation |
    | SessionStart | session-start, grepai-session | Initialization, grepai auto-start |
    | PreToolUse | grepai-reminder | Search reminder |
    | PermissionRequest | permission-guard | Auto-allow safe writes |
  </TabItem>
</Tabs>
