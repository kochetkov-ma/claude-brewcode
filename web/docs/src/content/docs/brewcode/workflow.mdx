# Workflow

Brewcode is a lean toolkit, not a pipeline. There are two skills and a set of agents.
This page shows how they fit together: write a structured spec with
[/brewcode:spec](/brewcode/skills/spec/), wire up semantic code search with
[/brewcode:grepai](/brewcode/skills/grepai/), then delegate the actual work to the
specialized [agents](/brewcode/agents/).

## High-level overview

```
/brewcode:spec            /brewcode:grepai           agents
      |                         |                       |
  Research                  Semantic index         Implement / test /
  (5-10 agents)             (Ollama + bge-m3)      review / architect
      |                         |                       |
      v                         v                       v
   SPEC.md                grepai_search           working changes
```

## Specification (/brewcode:spec)

The [spec](/brewcode/skills/spec/) skill creates a structured task specification through parallel
codebase research and a short user dialog.

<Steps>
  <li class="step step-primary">
    <div>
      <strong>Input parsing</strong>
      <p>Determines the input type: a text description or a path to a requirements file.
      Supports the <code>-n</code> (--noask) flag for autonomous mode.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Clarifying questions</strong>
      <p>3-5 questions via AskUserQuestion across categories: Scope (what's in/out),
      Constraints (libraries, compatibility), Edge cases (concurrency, null inputs).
      Skipped in <code>-n</code> mode.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Parallel research</strong>
      <p>5-10 agents are launched <strong>in a single message</strong> (in parallel).
      Each agent analyzes its area: Controllers, Services, DB/Repos, Tests, Config, Docs.</p>
      <pre><code class="language-text">Task(subagent_type="developer", prompt="Analyze services...")<br/>Task(subagent_type="tester", prompt="Analyze test patterns...")<br/>Task(subagent_type="reviewer", prompt="Analyze quality...")<br/>Task(subagent_type="Explore", prompt="Find library docs...")</code></pre>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Consolidation into SPEC.md</strong>
      <p>Agent results are merged with deduplication and written to a structured
      <code>SPEC.md</code> document.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Specification review</strong>
      <p>A reviewer checks completeness, consistency, feasibility, and risks.
      Fix loop: continues while critical/major issues exist (max 3 iterations).</p>
    </div>
  </li>
</Steps>

## Semantic search (/brewcode:grepai)

The [grepai](/brewcode/skills/grepai/) skill sets up AI-powered semantic code search backed by
Ollama and the bge-m3 embedding model. Once configured, agents call `grepai_search` to find
relevant code by intent rather than by literal keyword.

<Steps>
  <li class="step step-primary">
    <div>
      <strong>Configure</strong>
      <p>The <a href="/brewcode/agents/bc-grepai-configurator/">grepai configurator</a> agent
      generates a <code>.grepai/</code> config tailored to the project's languages and layout.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Index and watch</strong>
      <p>The <code>grepai-session</code> hook auto-starts the watch process on session start
      when <code>.grepai/</code> exists, keeping the index fresh.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Search by intent</strong>
      <p>Use <code>grepai_search</code> with a short English query (3-7 words describing intent).
      The <code>grepai-reminder</code> hook nudges agents to prefer it over plain text search.</p>
    </div>
  </li>
</Steps>

<Callout type="tip" title="Query tips">
  Search by intent, not syntax. Prefer <code>"validate credentials"</code> over
  <code>"validateUser"</code>. Three to seven English words works best.
</Callout>

## Delegation to agents

Brewcode ships nine specialized agents. After a spec exists and search is configured,
hand the real work to them.

| Need | Agent |
|------|-------|
| Implement a feature or fix a bug | [developer](/brewcode/agents/developer/) |
| Write or debug tests | [tester](/brewcode/agents/tester/) |
| Review code, security, performance | [reviewer](/brewcode/agents/reviewer/) |
| Analyze architecture and patterns | [architect](/brewcode/agents/architect/) |
| Author a new skill | [skill-creator](/brewcode/agents/skill-creator/) |
| Author a new agent | [agent-creator](/brewcode/agents/agent-creator/) |
| Author a new hook | [hook-creator](/brewcode/agents/hook-creator/) |
| Write a shell script | [bash-expert](/brewcode/agents/bash-expert/) |

<Callout type="tip" title="Run agents in parallel">
  Independent agents launched in a single message run concurrently, with isolated context.
  See [Agents](/brewcode/agents/) for the full roster, models, and tools.
</Callout>
