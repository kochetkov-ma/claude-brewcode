# Skills

Skills (recipes) are slash commands that define *what* to do. Each skill is described in a `SKILL.md` file
with frontmatter metadata and instructions for agents. Brewcode includes 17 skills covering
the full task lifecycle -- from project analysis to deployment and E2E testing.

## Lifecycle

The recommended skill execution order for a full task cycle:

<Steps>
  <li>
    <div>
      <strong>Setup</strong>
      <p>Analyze project, generate templates, check prerequisites. <a href="/brewcode/skills/setup/">setup</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Spec</strong>
      <p>Create specification through research and user interaction. <a href="/brewcode/skills/spec/">spec</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Plan</strong>
      <p>Create execution plan from SPEC with phases and dependencies. <a href="/brewcode/skills/plan/">plan</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Start</strong>
      <p>Execute task with infinite context -- automatic session handoff at context limits. <a href="/brewcode/skills/start/">start</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Rules</strong>
      <p>Sync learned knowledge to project rules for future sessions. <a href="/brewcode/skills/rules/">rules</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Teardown</strong>
      <p>Clean up plugin files when task is complete. <a href="/brewcode/skills/teardown/">teardown</a></p>
    </div>
  </li>
</Steps>

<Callout type="tip" title="Skill Management">
  Use <a href="/brewcode/skills/skills/">/brewcode:skills</a> to list, create, or improve skills.
  The <a href="/brewcode/agents/skill-creator/">Skill Creator</a> agent handles the creation process.
</Callout>

## Skill dependencies

```
setup --> spec --> plan --> start --> rules
  |        |                 |        |
  |        +-- convention    |        +-- teardown
  |        |                 |
  |        +-- grepai        +-- teams
  |
  +-- (Phase 0: prerequisites)
```

<Callout type="note" title="Dynamic Teams — New">
  <code>/brewcode:teams</code> creates 5-20 project-specific agents with self-selection protocol,
  performance tracking, and automatic tuning. Run after <code>/brewcode:setup</code> to build
  a team adapted to your project's domains and architecture.
</Callout>

<Callout type="note" title="Mode Switcher">
  The <code>/brewcode:skills create</code> command can generate <strong>Mode Switcher</strong> skills —
  a special skill type that toggles persistent behavioral modes for the entire session.
  Mode instructions are injected into every prompt via hooks and survive auto-compaction.
  Keywords like "mode", "toggle", or "persistent" in the create prompt trigger
  the Mode Switcher workflow automatically.
</Callout>
