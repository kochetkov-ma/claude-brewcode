# Skills

Skills (recipes) are slash commands that define *what* to do. Each skill is described in a `SKILL.md` file
with frontmatter metadata and instructions for agents. Brewcode includes 9 skills covering
the full task lifecycle -- from project analysis to E2E testing.

On top of those, [`/brewcode:review`](/brewcode/skills/review/) is a **special dynamic skill** — it is not shipped with the plugin, it is **generated into every target project** by [`/brewcode:superreview`](/brewcode/skills/superreview/), tailored to the project's tech stack and agents. See the [review skill page](/brewcode/skills/review/) for details.

## Lifecycle

The recommended skill execution order for a standard project cycle:

<Steps>
  <li>
    <div>
      <strong>Spec</strong>
      <p>Create specification through research and user interaction. <a href="/brewcode/skills/spec/">spec</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Superreview</strong>
      <p>Generate a per-project deep-review skill tailored to your tech stack and agents. <a href="/brewcode/skills/superreview/">superreview</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Rules</strong>
      <p>Sync learned knowledge to project rules for future sessions. <a href="/brewcode/skills/rules/">rules</a></p>
    </div>
  </li>
  <li>
    <div>
      <strong>Teams (optional)</strong>
      <p>Create 5-20 project-specific agents with self-selection and performance tracking. <a href="/brewcode/skills/teams/">teams</a></p>
    </div>
  </li>
</Steps>

<Callout type="tip" title="Skill Management">
  Use <a href="/brewcode/skills/skills/">/brewcode:skills</a> to list, create, or improve skills.
  The <a href="/brewcode/agents/skill-creator/">Skill Creator</a> agent handles the creation process.
</Callout>

## Skill dependencies

```
spec --> superreview --> review (dynamic)
  |
  +-- grepai (optional)
  +-- convention (optional)
  +-- rules
  +-- teams (optional)
```

<Callout type="note" title="Dynamic Teams — New">
  <code>/brewcode:teams</code> creates 5-20 project-specific agents with self-selection protocol,
  performance tracking, and automatic tuning. Run <code>/brewcode:teams</code> to build
  a team adapted to your project's domains and architecture.
</Callout>
