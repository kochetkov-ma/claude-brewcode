# Hooks

Hooks (processes) are scripts attached to Claude Code lifecycle events. They initialize the session,
auto-start semantic search, nudge agents toward grepai, boost skill activation, and auto-allow safe
writes. Brewcode registers 5 hooks in `hooks.json`.

## Summary table

| # | Hook | Event | Matcher | Channel | Timeout | Purpose |
|---|------|-------|---------|---------|---------|---------|
| 1 | forced-eval | UserPromptSubmit | -- | additionalContext | 1s | Skill activation boost |
| 2 | session-start | SessionStart | -- | additionalContext | 3s | Session initialization, LATEST.md |
| 3 | grepai-session | SessionStart | -- | systemMessage | 5s | Auto-start grepai watch |
| 4 | grepai-reminder | PreToolUse | Bash | additionalContext | 1s | grepai_search reminder |
| 5 | permission-guard | PermissionRequest | Edit\|Write\|MultiEdit\|Bash | hookSpecificOutput | 1s | Auto-allow safe .claude/ writes |

## Execution flow

```
UserPromptSubmit
  └── forced-eval.mjs          [SKILL?] Check available skills

SessionStart
  ├── session-start.mjs        Session ID, LATEST.md symlink, permission_mode
  └── grepai-session.mjs       Auto-start grepai watch

PreToolUse:Bash
  └── grepai-reminder.mjs      "USE grepai_search FIRST"

PermissionRequest:Edit|Write|MultiEdit|Bash
  └── permission-guard.sh      Auto-allow safe .claude/ and /tmp paths
```

## I/O protocol

All hooks follow a unified protocol:

1. Read JSON from stdin (via `readStdin()`)
2. Receive fields: `session_id`, `cwd`, `source` (SessionStart), `tool_input` (PreToolUse), `permission_mode` (all events)
3. Output JSON to stdout (via `output()`)
4. Write logs to stderr and to the file `.claude/tasks/logs/brewcode.log`

**Shared utilities:**
- `hooks/lib/utils.mjs` -- I/O, configuration, logging

## Detailed description

### 1. forced-eval.mjs

<Badge variant="primary" text="UserPromptSubmit" /> <Badge variant="secondary" text="additionalContext" />

Boosts skill activation from ~20-50% to ~84%. Intercepts every user prompt
and adds a reminder to check available skills.

| Parameter | Value |
|-----------|-------|
| Event | UserPromptSubmit |
| Channel | additionalContext |
| Timeout | 1000 ms |

**What it does:**
- Receives the user prompt
- Adds a prefix: `[SKILL?] Check available skills. If one matches this request, use Skill tool before responding.`
- Injects the skill-check reminder (and a light delegation hint) via `additionalContext` -- UserPromptSubmit cannot rewrite the prompt
- Output is capped at ~9000 chars (10K disk-spill safety, CC 2.1.174+)

**When it fires:** On every user input, except events with an incorrect `hook_event_name`.

### 2. session-start.mjs

<Badge variant="primary" text="SessionStart" /> <Badge variant="secondary" text="additionalContext" />

Initializes the session, logs the session ID, manages Plan Mode symlinks, and reports the active `permission_mode` in the system message for audit.

| Parameter | Value |
|-----------|-------|
| Event | SessionStart |
| Channel | additionalContext |
| Timeout | 3000 ms |

The hook reads `permission_mode` from the hook payload and appends it to `systemMessage` so every session records its trust level (default / plan / bypassPermissions). Output is capped at ~9000 chars (10K disk-spill safety, CC 2.1.174+).

**Logic by session source:**

| Source | Behavior |
|--------|----------|
| `init` | Log session_id |
| `resume` | Log session_id |
| `clear` | Create symlink LATEST.md -> newest plan |

**LATEST.md symlink:**
1. Checks `&#126;/.claude/plans/` for `.md` files
2. Picks the newest one (by mtime)
3. If the file is less than 60 seconds old -- creates `.claude/plans/LATEST.md` -> `&#126;/.claude/plans/<newest>.md`
4. Used by Plan Mode to resolve the latest plan

### 3. grepai-session.mjs

<Badge variant="primary" text="SessionStart" /> <Badge variant="secondary" text="systemMessage" />

Automatically starts grepai watch when entering a project with a configured `.grepai/`.

| Parameter | Value |
|-----------|-------|
| Event | SessionStart |
| Channel | systemMessage |
| Timeout | 5000 ms |

**Logic:**
- If `.grepai/` does not exist -- skips silently
- If `.grepai/` exists -- checks ollama, index, watch process, MCP
- If watch is not running -- starts it automatically
- Reports status via `systemMessage`

<Callout type="note" title="Safety">
  This hook never blocks session start. All errors are handled as informational.
  Platform: macOS/Linux only (Windows not supported due to missing pgrep).
</Callout>

### 4. grepai-reminder.mjs

<Badge variant="primary" text="PreToolUse:Bash" /> <Badge variant="secondary" text="additionalContext" />

Reminds to use `grepai_search` before running a search command via Bash.

| Parameter | Value |
|-----------|-------|
| Event | PreToolUse |
| Matcher | Bash |
| Channel | additionalContext |
| Timeout | 1000 ms |

<Callout type="note" title="Why Bash, not Glob/Grep">
  On macOS Claude Code (v2.1.117+) the native Grep/Glob tools were removed -- search now
  runs through the Bash tool (grep -> ugrep, find -> bfs, rg -> rg). The matcher is
  <code>Bash</code> and the hook gates on a search-command regex so the reminder only
  appears for actual searches.
</Callout>

**Throttling:** The reminder is shown at most once per 60 seconds (via timestamp file `.grepai/.reminder-ts`).

**Condition:** Only fires if `.grepai/` and `.grepai/index.gob` exist.

### 5. permission-guard.sh

<Badge variant="primary" text="PermissionRequest" /> <Badge variant="secondary" text="hookSpecificOutput" />

Auto-allows write operations to safe `.claude/` subdirectories and temp paths,
eliminating manual permission prompts. Written in bash for minimal overhead.

| Parameter | Value |
|-----------|-------|
| Event | PermissionRequest |
| Matcher | Edit\|Write\|MultiEdit\|Bash |
| Channel | hookSpecificOutput (decision: allow) |
| Timeout | 1000 ms |

**Allowed paths for Edit/Write/MultiEdit:**

| Path pattern | Example |
|--------------|---------|
| `.claude/tasks/*` | `.claude/tasks/20260301_auth_task/SPEC.md` |
| `.claude/tmp/*` | `.claude/tmp/scratch.md` |
| `.claude/reports/*` | `.claude/reports/review.md` |
| `.claude/rules/*` | `.claude/rules/best-practice.md` |
| `.claude/skills/*` | `.claude/skills/custom/SKILL.md` |
| `.claude/scripts/*` | `.claude/scripts/build.sh` |
| `.claude/agents/*` | `.claude/agents/reviewer.md` |
| `.claude/hooks/*` | `.claude/hooks/hooks.json` |
| `.claude/private/*` | `.claude/private/notes.md` |
| `.claude/convention/*` | `.claude/convention/patterns.md` |
| `.claude/plans/*` | `.claude/plans/LATEST.md` |
| `.claude/settings.json` | Project settings |
| `.claude/TASK.md`, `.claude/CLAUDE.md` | Quick-ref files |
| `/tmp/*`, `/private/tmp/*` | System temp directories |

**Bash tool handling:**

The hook applies additional safety checks for Bash commands:
- **Network commands blocked:** `curl`, `wget`, `ssh`, `scp`, `rsync`, `nc`, `ftp`, `sftp`
- **Dangerous redirections blocked:** writes to `/dev/`, `/etc/`, `eval`, `exec`, `source`
- **`rm` restricted:** only allowed within `.claude/tasks/`, `.claude/tmp/`, and `/tmp/`
- **Path extraction:** all `.claude/` and temp path tokens are verified against the allow-list

<Callout type="caution" title="Global config excluded">
  The hook explicitly skips the global <code>&#126;/.claude/</code> directory.
  Only project-level <code>.claude/</code> paths receive auto-allow.
  Writes to <code>&#126;/.claude/CLAUDE.md</code> or <code>&#126;/.claude/rules/</code> still require manual permission.
</Callout>

**Decision logic:**

| State | Action |
|-------|--------|
| Path in allowed list | `allow` -- bypass permission prompt |
| Path outside allowed list | `{}` -- fall through to default prompt |
| No file_path (non-matching tool) | `{}` -- fall through |

## Configuration

Hooks read settings from `.claude/tasks/cfg/brewcode.config.json`:

| Parameter | Default value | Description |
|-----------|---------------|-------------|
| `logging.level` | `info` | Logging level (error/warn/info/debug/trace) |
