# Teams

<Badge variant="primary" text="opus" /> <Badge variant="accent" text="killer feature" />

## Quick reference

| Field | Value |
|-------|-------|
| Command | `/brewcode:teams` |
| Arguments | `create <name> [prompt]`, `status <name>`, `update <name>`, `cleanup <name>` |
| Model | opus |
| Tools | Read, Write, Edit, Glob, Grep, Bash, Task, AskUserQuestion |

## Modes

<Tabs>
  <TabItem label="Create">
    | Step | Action |
    |------|--------|
    | 1 | Analyze project with 3-5 Explore agents |
    | 2 | Propose 3 team variants (Minimal/Balanced/Maximum) |
    | 3 | User selects variant and default model |
    | 4 | Create agents in batches of 3-4 via agent-creator |
    | 5 | Set up `team.md` + `trace.jsonl` in `.claude/teams/` |
  </TabItem>
  <TabItem label="Status">
    | Step | Action |
    |------|--------|
    | 1 | Read `trace.jsonl` via `trace-ops.sh read` |
    | 2 | Calculate per-agent success rates |
    | 3 | Classify health: green/yellow/red |
    | 4 | Report issues and insights from trace data |
  </TabItem>
  <TabItem label="Update">
    | Step | Action |
    |------|--------|
    | 1 | Analyze performance data |
    | 2 | Identify underperformers |
    | 3 | Tune instructions or replace agents |
    | 4 | Update team.md roster |
  </TabItem>
  <TabItem label="Cleanup">
    | Step | Action |
    |------|--------|
    | 1 | Archive old trace data to `trace-archive.jsonl` |
    | 2 | Remove inactive agents |
    | 3 | Reset trace cursor |
  </TabItem>
</Tabs>

## Review and Fix Pipeline

After agents are created, a quality pipeline validates and hardens the team:

<Steps>
  <li>
    <div>
      <strong>Quorum Review</strong> <Badge variant="secondary" text="C5" />
      <p>3 independent reviewer agents (domain experts matching the team's scope) analyze every created agent in parallel. Each reviewer checks: instruction quality, domain accuracy, tool selection, description triggers, model fit.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Consensus Filter</strong> <Badge variant="secondary" text="C6" />
      <p>Issues confirmed by 2/3 reviewers pass the quorum threshold. Minor issues (style, cosmetic) are logged but skipped. Only critical and important findings proceed.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Verification</strong> <Badge variant="secondary" text="C7" />
      <p>An additional verification agent cross-checks each confirmed finding against the actual agent file. False positives are filtered out. Verified issues get a severity tag: critical (broken), important (degraded), minor (cosmetic).</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Fix</strong> <Badge variant="secondary" text="C8" />
      <p>agent-creator fixes all critical and important verified issues. Minor issues are skipped. Each fix is applied to the agent file directly.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Re-verify</strong> <Badge variant="secondary" text="C9" />
      <p>Verification agent re-checks every fix. If a fix introduced a regression, it goes back to the Fix step (max 2 cycles). Only after clean re-verification does the pipeline complete.</p>
    </div>
  </li>
</Steps>

<Callout type="tip" title="Pipeline Control">
  The review pipeline runs automatically after agent creation. To skip it, use <code>--skip-review</code>.
  To run it separately on an existing team: <code>/brewcode:teams update &lt;name&gt; --review</code>.
</Callout>

## Task Acceptance Protocol

<CardGrid>
  <Card title="Domain Check" icon="shield">
    Agent evaluates: "Is this my domain?" If not, refuses and suggests a better-suited colleague from the team roster.
  </Card>
  <Card title="Duplicate Check" icon="search">
    Agent checks: "Has this already been done?" If yes, refuses and links to the existing result in trace.jsonl.
  </Card>
  <Card title="Best Candidate" icon="people">
    Agent considers: "Is a colleague better suited?" If yes, refuses and redirects to the recommended agent.
  </Card>
</CardGrid>

## Framework Files

<CardGrid>
  <Card title="team.md" icon="list">
    Agent roster: names, domains, missions, health status. Source of truth for the team composition.
  </Card>
  <Card title="trace.jsonl" icon="chart">
    Unified JSONL log: tasks (took/refused/completed/failed), issues (by severity), insights (by category). Agents write via `trace-ops.sh add` (Bash append, ~96% token savings). Session-scoped via SID.
  </Card>
</CardGrid>

## Agent Health

| Health | Criteria | Action |
|--------|----------|--------|
| Green | >70% success rate, active in recent tasks | No action needed |
| Yellow | 30-70% success rate or many refusals | Review instructions, consider tuning |
| Red | &lt;30% success rate or inactive | Replace or remove via update mode |

<Callout type="note" title="Integration with Core Skills">
  Team agents are automatically preferred by <code>/brewcode:spec</code>, <code>/brewcode:plan</code>, <code>/brewcode:start</code>
  when matching domains are found. Priority: team agent &rarr; project agent &rarr; plugin agent.
  Agents can refuse tasks and suggest a better-suited colleague.
</Callout>
