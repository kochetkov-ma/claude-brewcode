# Clean up brewcode scaffolding when you're done.

<Callout type="caution">
**Teardown deletes brewcode project files permanently.** Templates, configs, logs, plan directories, and grepai indexes are removed. The operation asks for confirmation before touching anything — unless `--dry-run` is passed.
</Callout>

<Callout type="tip">
**Task directories and user rules are never touched.** Everything under `.claude/tasks/*_task/` and `.claude/rules/` survives teardown unconditionally. Use `--dry-run` first to preview exactly what would be removed.
</Callout>

## Quick reference

| Field | Value |
|-------|-------|
| Command | `/brewcode:teardown` |
| Arguments | `[--dry-run]` |
| Model | haiku |
| Context | fork |
| Tools | Bash, Read, AskUserQuestion |

## When to use

- **Project wrap-up** — the task batch is done and the brewcode scaffolding (рецепты-шаблоны, скиллы) is no longer needed in the repo
- **Fresh start** — you want to re-run `/brewcode:setup` on a clean slate without leftover configs
- **Disk cleanup** — `.grepai/` indexes or log directories have grown large and are no longer useful
- **Preview before commit** — `--dry-run` shows exactly which files would go, without committing to removal

## Examples

```bash
# Preview removals without deleting anything
/brewcode:teardown --dry-run

# Full teardown (prompts for confirmation)
/brewcode:teardown
```

```bash
# Natural language also triggers the skill
"clean up brewcode files"
"tear down brewcode setup"
```

## Flow

<Steps>
  <li>
    <div>
      <strong>Receive arguments</strong>
      <p>The skill reads <code>$ARGUMENTS</code>. If <code>--dry-run</code> is present, skip confirmation and go straight to listing. Otherwise proceed to the confirmation step.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Confirm (full mode only)</strong>
      <p>AskUserQuestion: <em>"This will delete brewcode project files (templates, configs, logs, plans). Task directories are preserved. Proceed?"</em> — answering No aborts with no changes made.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Run teardown script</strong>
      <p>Executes <code>teardown.sh</code> from <code>$CLAUDE_SKILL_DIR/scripts/</code>. In dry-run mode the script only prints; in full mode it deletes. The skill stops with an error if the script is missing or not executable.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Print removal report</strong>
      <p>Each removed path is listed with a ✅. Paths that were skipped (not found or preserved) are listed with ⏭️. No silent deletions.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Done</strong>
      <p>The project directory is clean. Run <code>/brewcode:setup</code> again at any time to re-initialise scaffolding.</p>
    </div>
  </li>
</Steps>

## Options

| Option | Behavior |
|--------|----------|
| `--dry-run` | List files that would be deleted — no confirmation prompt, no changes |
| (none) | Full removal after explicit user confirmation |

<details>
<summary>What teardown removes vs. keeps</summary>

**Removed**

| Path | Contents |
|------|----------|
| `.claude/tasks/templates/` | PLAN.md, SPEC.md, KNOWLEDGE.jsonl starters |
| `.claude/tasks/cfg/` | `brewcode.config.json`, `brewcode.state.json` |
| `.claude/tasks/logs/` | Execution logs |
| `.claude/plans/` | Intermediate plan files |
| `.grepai/` | Semantic search indexes |
| `.claude/skills/brewcode-review/` | Quorum review working files |

**Always preserved**

| Path | Reason |
|------|--------|
| `.claude/tasks/*_task/` | Completed task work — your artifacts |
| `.claude/rules/` | User rules — not managed by brewcode |

</details>
