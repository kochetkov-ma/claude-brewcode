# Leaked credentials are invisible until they're not.

<Callout type="caution">
**Secrets committed to git are permanent — even after deletion.** A single `password=prod123` in a config file, a hardcoded AWS key in a test fixture, a connection string left in a migration script. Once pushed, the history retains it. `/brewtools:secrets-scan` finds these before they become incidents.
</Callout>

<Callout type="tip">
**10 parallel agents, one command.** The skill (рецепт) splits your git file list into 10 chunks, runs haiku agents in parallel, merges results, ranks by severity, and writes a structured report — all without leaving your Claude session. Add `--fix` to remediate interactively.
</Callout>

## Quick reference

| Field | Value |
|-------|-------|
| Command | `/brewtools:secrets-scan` |
| Arguments | `[--fix]` — no args = scan only, `--fix` = interactive remediation |
| Model | sonnet |
| Context | session |
| Tools | Read, Task, Write, Bash, AskUserQuestion |

## When to use

- **Pre-push audit** — run before opening a PR or pushing to a remote branch
- **Open-source prep** — before making a private repo public, scan the full history surface
- **CI gate** — catch secrets that slipped past `.gitignore` rules
- **Post-incident** — verify no additional credentials are exposed after a breach
- **Onboarding** — audit a legacy codebase you've just inherited

## Examples

```bash
# Scan only — full repo, severity-ranked report
/brewtools:secrets-scan

# Scan and remediate interactively (CRITICAL + HIGH findings prompted)
/brewtools:secrets-scan --fix

# Natural language triggers
"scan for leaked credentials"
"security audit, find any hardcoded keys"
"check for secrets before I open-source this repo"
```

Expected output after scan:

```
## Secrets Scan Complete

| Metric   | Value |
|----------|-------|
| Files    | 342   |
| CRITICAL | 1     |
| HIGH     | 3     |
| MEDIUM   | 7     |
| LOW      | 12    |

Report: .claude/reports/20260417-143022_secrets-scan/report.md
```

## Flow

<Steps>
  <li>
    <div>
      <strong>Setup</strong>
      <p>Verifies the working directory is a git repository, collects the full file list via <code>git ls-files</code>, creates a timestamped report directory under <code>.claude/reports/</code>.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Split &amp; launch 10 agents</strong>
      <p>Divides the file list into 10 equal chunks (ceil division). Spawns 10 <code>general-purpose haiku</code> agents in a single parallel Task call — each agent reads its assigned files and applies detection patterns.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Merge results</strong>
      <p>Collects JSON responses from all 10 agents. Deduplicates findings by <code>path+line</code>, merges scanned and skipped file lists, sorts findings: CRITICAL → HIGH → MEDIUM → LOW.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Generate report</strong>
      <p>Writes <code>&#123;DIR&#125;/report.md</code> with a summary table, per-severity finding tables (file, line, matched content, description), per-agent stats, and a full file inventory including skipped files with skip reasons.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Display summary</strong>
      <p>Prints a compact table of severity counts and the report path directly in the session. No need to open the file to see the headline result.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Fix mode</strong>
      <p>Triggered by <code>--fix</code> flag or automatically when CRITICAL/HIGH findings exist. Walks through each finding with <code>AskUserQuestion</code>: fix inline, move to env var, add file to <code>.gitignore</code>, mark as false positive, or skip.</p>
    </div>
  </li>
</Steps>

<details>
<summary>Detection rules</summary>

**What gets flagged:**

| Category | Patterns |
|----------|---------|
| Passwords | `password`, `passwd`, `secret`, `pwd` followed by `=` or `:` and a value |
| API keys | `api_key`, `access_key`, `apikey`, `api_secret` |
| Tokens | `token`, `bearer`, `auth_token`, `access_token` |
| AWS credentials | `AKIA[0-9A-Z]{16}`, `aws_secret`, `aws_access_key` |
| Database URLs | JDBC, MongoDB, MySQL, PostgreSQL connection strings with embedded credentials |
| Private keys | `-----BEGIN ... PRIVATE KEY-----`, `client_secret`, `encryption_key` |

**Severity levels:**

| Level | Criteria |
|-------|----------|
| CRITICAL | Real credentials, private keys, database connection strings with passwords |
| HIGH | Real API keys or tokens, AWS credentials |
| MEDIUM | Suspicious hardcoded values that may be secrets |
| LOW | Placeholder values (`changeme`, `YOUR_KEY`, `xxx`, `dummy`) |

**Automatically skipped:**

Environment variable references (`process.env.*`, `${VAR}`, `os.getenv()`), common placeholders, documentation comments, binary files.

**Fix mode options:**

| Option | Action |
|--------|--------|
| Fix | Delete or replace the secret inline |
| Move to env var | Extract value into an environment variable reference |
| Add to .gitignore | Append the file path to `.gitignore` |
| Mark false positive | Acknowledge and skip — logged in report |
| Skip | Leave as-is, move to next finding |

</details>
