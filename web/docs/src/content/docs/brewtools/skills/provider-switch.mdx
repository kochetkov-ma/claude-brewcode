# Provider Switch

<Badge variant="accent" text="new" /> <Badge variant="primary" text="opus" />

Switch Claude Code between alternative API providers. Run one alias, start Claude — connected. Run `claude-max` — back on your subscription.

<CardGrid>
  <Card title="Z.ai / GLM" icon="star">
    #1 SWE-bench Pro, pay-per-token. Best raw coding performance per dollar.
  </Card>
  <Card title="Qwen" icon="document">
    1M context window, free preview available. Ideal for large-codebase exploration.
  </Card>
  <Card title="MiniMax" icon="money">
    Cheapest per token. Solid for high-volume summarization and text tasks.
  </Card>
  <Card title="OpenRouter" icon="sync">
    200+ models, any provider. Route to GPT-4o, Gemini, Llama, or any hosted model.
  </Card>
</CardGrid>

## Quick Reference

| Field | Value |
|-------|-------|
| Command | `/brewtools:provider-switch` |
| Arguments | `[provider]` or `setup / help` |
| Modes | status, setup, help, glm, qwen, minimax, openrouter |
| Model | opus |

## Quick Start

```bash
# Check current provider status
/brewtools:provider-switch

# Set up all providers interactively
/brewtools:provider-switch setup

# Configure just GLM
/brewtools:provider-switch glm

# Get help on switching
/brewtools:provider-switch help
```

## Switching Providers

<Callout type="tip">
  **How to switch.** Run the alias (`claude-glm`), then start `claude`. To return to your Anthropic subscription: run `claude-max` then `claude`. Each alias sets environment variables for one provider — they are fully isolated and do not affect each other.
</Callout>

Each alias exports the correct `ANTHROPIC_BASE_URL`, `ANTHROPIC_API_KEY` or `ANTHROPIC_AUTH_TOKEN`, and optionally a default model. Sourcing a new alias overwrites the previous export.

```bash
# Switch to GLM
claude-glm && claude

# Switch back to Anthropic subscription
claude-max && claude
```

## Technical Details

### Arguments

| Argument | Description |
|----------|-------------|
| _(none)_ | Show current status table |
| `setup` | Interactive setup for all providers |
| `help` | Print switching cheat sheet |
| `glm` | Configure Z.ai/GLM only |
| `qwen` | Configure Qwen only |
| `minimax` | Configure MiniMax only |
| `openrouter` | Configure OpenRouter only |

### Modes

| Mode | Trigger | Behavior |
|------|---------|----------|
| `status` | No argument | Read `~/.zshrc`, display provider table |
| `setup` | `setup` | Full interactive setup, all providers |
| `help` | `help` | Print alias cheat sheet |
| Provider mode | `glm` / `qwen` / `minimax` / `openrouter` | Single-provider setup flow |

### Context window 1M hack

Claude Code defaults to 200K context for non-Anthropic providers. Adding `[1m]` to the model name (e.g., `qwen3.6-plus[1m]`) forces 1M context detection. The skill applies this suffix automatically for models that support it — ensuring Claude Code does not autocompact prematurely.

### Auth differences

| Provider | Variable | Notes |
|----------|----------|-------|
| Z.ai / GLM | `ANTHROPIC_API_KEY` | Z.ai mirrors Anthropic key format |
| Qwen | `ANTHROPIC_AUTH_TOKEN` | DashScope token format |
| MiniMax | `ANTHROPIC_AUTH_TOKEN` | Bearer token |
| OpenRouter | `ANTHROPIC_AUTH_TOKEN` | `sk-or-...` prefix |

<Callout type="note">
  The skill never stores keys in project files. All secrets go into shell aliases in `~/.zshrc`, which is gitignored by definition.
</Callout>
