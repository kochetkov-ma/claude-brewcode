# Plugin Update

<Badge variant="primary" text="sonnet" /> <Badge variant="secondary" text="session" />

## Quick reference

| Field | Value |
|-------|-------|
| Command | `/brewtools:plugin-update` |
| Arguments | `[check\|update\|all]` -- no args = `all` (check + update) |
| Model | sonnet |
| Context | session |
| Tools | Bash, AskUserQuestion |

Checks the installed version of every brewcode plugin against the latest available release,
then runs the full update chain and prompts you to reload. Designed to be run without arguments
for a zero-friction update experience.

## Modes

| Argument | What it does |
|----------|-------------|
| _(none)_ | Same as `all`: check versions, then update if a newer version is found |
| `check` | Report installed and latest versions for each plugin; no changes made |
| `update` | Run the update chain unconditionally without a version check first |
| `all` | Check versions, then update all plugins if any are out of date |

## Phases

**Phase 1 -- Version check** (skipped when argument is `update`)

Reads the installed version from the local plugin cache and compares it against the latest GitHub release tag. Outputs a table showing each plugin's current and latest version.

**Phase 2 -- Update** (skipped when argument is `check`)

Runs the update chain in order, showing full output for each command:

```bash
claude plugin marketplace update claude-brewcode
claude plugin update brewcode@claude-brewcode
claude plugin update brewdoc@claude-brewcode
claude plugin update brewtools@claude-brewcode
claude plugin update brewui@claude-brewcode
```

**Phase 3 -- Reload**

After a successful update, prompts you to run `/reload-plugins`. If that command is unavailable in your version of Claude Code, instructs you to type `exit` and run `claude` again.

## Usage examples

```bash
# Full check-and-update (most common)
/brewtools:plugin-update

# Only check versions, no changes
/brewtools:plugin-update check

# Update unconditionally without a version check
/brewtools:plugin-update update
```

## When Claude refuses to run update commands

If Claude skips or summarizes the update commands instead of running them, use the coercive prompt from the [FAQ](/faq/) -- it forces step-by-step execution with full output.

<Callout type="tip" title="Scheduled updates">
  Run <code>/brewtools:plugin-update check</code> at the start of a session to see whether your plugins are current before starting long-running tasks.
</Callout>
