# Manager — codeword-triggered Manager mode

<Badge variant="default" text="sonnet" /> <Badge variant="accent" text="hook" />

<Callout type="tip">
**Type `++m` anywhere in a prompt — the Manager block injects itself for that turn.** No mode change, no command to run. The hook (`manager-prompt.mjs`, UserPromptSubmit) detects the codeword, reads the effective state, and appends the full Manager prompt as `additionalContext`. Zero overhead when the codeword is absent or the skill is off.
</Callout>

## Quick reference

| Item | Value |
|------|-------|
| Skill command | `/brewtools:manager` |
| Codewords | `++m` — full Manager block; `++mp` — Manager + Plan Mode addon |
| Default state | **ON** (`enabled: true`, `mode: "full"`) |
| Hook event | `UserPromptSubmit` — fires on every user message, checks for codeword |
| Scope | project (`.claude/brewtools/manager/`) → global (`~/.claude/manager/`) → plugin default |
| Prompt resolution | project override → global override → plugin `references/<mode>.md` |
| Model | sonnet |

## When to use

| Situation | Invocation |
|-----------|-----------|
| Delegate one task to Manager for that turn only | Type `do X ++m` — no persistent state change |
| Start a full Manager-mode plan with parallel tasks | Type `solve Y ++mp` |
| Enable Manager mode persistently | `/brewtools:manager on` or `включи` |
| Switch to Plan Mode addon globally | `/brewtools:manager mode planmode` |
| Check effective state + prompt source | `/brewtools:manager status` or `что сейчас` |
| Customize the Manager prompt text | `/brewtools:manager edit full` or `поправь промт` |
| Run a task inline as Manager right now | `/brewtools:manager <task description>` |

## Examples

**Codeword in a normal prompt — Manager block injects for this turn only:**
```
разбери архитектуру модуля auth и предложи план рефакторинга ++m
```

**Plan Mode codeword — adds TaskGraph + English-plan block:**
```
migrate the database layer to a new ORM ++mp
```

**Enable globally:**
```
включи глобально
/brewtools:manager on --scope global
```

**Check current state:**
```
что сейчас
/brewtools:manager status
```

Expected status output:
```
# Manager — status
enabled:       true   (state source: project)
active mode:   full
codewords:     ++m → full · ++mp → planmode
prompt source: full=default  planmode=default

--- injected block: ++m (full) ---
<full Manager block text>

--- injected block: ++mp (planmode) ---
<planmode Manager block text>
```

**Switch to Plan Mode addon:**
```
переключи на planmode
/brewtools:manager mode planmode
```

**Edit the Manager prompt (copies default into writable override):**
```
поправь промт
/brewtools:manager edit full
```

**Inline-run — prepend Manager block and execute the request now:**
```
/brewtools:manager build a task graph for refactoring the payment service
```

**Disable:**
```
выключи
/brewtools:manager off
```

## How it works

<Steps>
  <li>
    <div>
      <strong>UserPromptSubmit — codeword detection</strong>
      <p>On every user message, <code>manager-prompt.mjs</code> scans the prompt text. It tests for <code>++mp</code> before <code>++m</code> to avoid prefix collision. If neither codeword is present, the hook exits immediately — no file I/O, no overhead.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>State check — enabled guard</strong>
      <p>If a codeword is found, the hook reads the effective state: project <code>.claude/brewtools/manager/state.json</code> → global <code>~/.claude/manager/state.json</code> → hardcoded default <code>{"{"}enabled:true, mode:"full"{"}"}</code>. If <code>enabled === false</code>, the hook injects nothing and returns — fail-safe: any read error also skips injection.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Prompt resolution — three-level fallback</strong>
      <p>The hook resolves the prompt text for the detected mode (<code>full</code> or <code>planmode</code>): project override <code>.claude/brewtools/manager/prompts/&lt;mode&gt;.md</code> → global override <code>~/.claude/manager/prompts/&lt;mode&gt;.md</code> → plugin default <code>references/&lt;mode&gt;.md</code>. Project and global overrides survive plugin updates because they live outside the plugin cache.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>additionalContext injection — same turn</strong>
      <p>The resolved prompt text is returned as <code>additionalContext</code> for the current turn. The model receives the Manager block alongside the user message and applies it immediately. The prompt is not stripped from the user message — codewords remain visible in history for auditability.</p>
    </div>
  </li>
  <li>
    <div>
      <strong>Skill control plane — intent-driven</strong>
      <p>The <code>/brewtools:manager</code> skill handles persistent changes: <code>on/off</code> write state JSON (atomic tmp+rename via Node helpers), <code>mode</code> sets active mode, <code>edit</code> copies the plugin default into a writable project/global override, <code>reset</code> deletes the override. The <code>inline-run</code> fallthrough prepends the full Manager block and executes the user request in the current turn without touching state.</p>
    </div>
  </li>
</Steps>

## Actions

| Action | Command / natural language | Effect |
|--------|---------------------------|--------|
| `on` | `/brewtools:manager on`, `включи` | Set `enabled: true` in project state |
| `off` | `/brewtools:manager off`, `выключи` | Set `enabled: false` in project state |
| `status` | `/brewtools:manager status`, `что сейчас` | Print effective state, sources, prompt file paths |
| `mode full` | `/brewtools:manager mode full`, `полный режим` | Set `mode: "full"` — plain Manager block |
| `mode planmode` | `/brewtools:manager mode planmode`, `переключи на planmode` | Set `mode: "planmode"` — Manager + Plan Mode addon |
| `edit [full\|planmode]` | `/brewtools:manager edit`, `поправь промт` | Copy plugin default into project override for editing |
| `reset [full\|planmode]` | `/brewtools:manager reset` | Delete project override, revert to plugin default |
| `inline-run` | `/brewtools:manager <task>` | Prepend Manager block, execute task — no state change |

## Codewords

| Codeword | Injected content | Note |
|----------|-----------------|------|
| `++m` | Full Manager block (`references/full.md` or override) | Tested second — `++mp` takes priority |
| `++mp` | Full Manager block + Plan Mode addon (`references/planmode.md` or override) | Tested first to avoid prefix collision with `++m` |

Codewords can appear anywhere in the prompt — start, middle, or end. Surrounding text is preserved unchanged.

## State + prompt resolution

### State JSON shape

```json
{
  "enabled": true,
  "mode": "full"
}
```

Both fields are optional — missing fields fall back to the next scope in the chain.

### Resolution chain

| Priority | State source | Prompt override source |
|----------|-------------|----------------------|
| 1 (highest) | `.claude/brewtools/manager/state.json` | `.claude/brewtools/manager/prompts/<mode>.md` |
| 2 | `~/.claude/manager/state.json` | `~/.claude/manager/prompts/<mode>.md` |
| 3 (lowest) | Hardcoded `{"{"}enabled:true, mode:"full"{"}"}` | Plugin `references/<mode>.md` |

Project scope wins over global field-by-field — `enabled` and `mode` can each come from different scopes.

### `--scope` flag

All `on/off/mode/edit/reset` actions default to **project** scope. Pass `--scope global` to write globally:

```
/brewtools:manager on --scope global
включи глобально
```

<Callout type="note">
Global paths (`~/.claude/*`) are protected from direct Write/Edit tool access (Claude Code v3.4.70+). The skill writes them via bundled Node helpers that use atomic lock+tmp+rename, so edits are safe and survive interruptions.
</Callout>

### Plan Mode addon (`++mp`)

When `++mp` is detected, the hook injects **both** the full Manager block and the Plan Mode addon. The addon instructs the model to:

- Write the full TaskGraph in English into the plan before starting
- Mark tasks with parallel / critical-path indicators
- Output all task and plan text in English regardless of user prompt language
