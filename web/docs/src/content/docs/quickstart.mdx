# Quick Start

In 15 minutes you will go through the lean Brewcode cycle: write a task spec, wire up
semantic code search, then hand the work to a coding agent.
By the end you will have a structured specification and a fast way to explore any codebase.

<Callout type="note" title="Before you begin">
  Make sure Claude Code and the brewcode plugin are installed.
  If not, follow the instructions in <a href="/installation/">Installation</a>.
</Callout>

## Full workflow

<Steps>
  <li class="step step-primary">
    <div>
      <strong>Creating a specification</strong> <Badge variant="primary" text="/brewcode:spec" />
      <p>Open Claude Code in your project root. Describe the task in plain text or point to a requirements file:</p>
      <pre><code>/brewcode:spec "Implement JWT authorization with role-based access"</code></pre>
      <p>Or from a file:</p>
      <pre><code>/brewcode:spec ./docs/requirements/auth.md</code></pre>
      <p><strong>What happens:</strong> the plugin launches research agents in parallel.
      The agents analyze the codebase, find related code, dependencies, and patterns.
      Then it asks clarifying questions (use the <code>-n</code> flag for autonomous mode without questions).</p>
      <p><strong>Result:</strong> a <code>SPEC.md</code> file -- a structured specification with project context.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Semantic code search</strong> <Badge variant="primary" text="/brewcode:grepai" />
      <pre><code>/brewcode:grepai</code></pre>
      <p><strong>What happens:</strong> the skill sets up grepai semantic search over your repository
      (it can install ollama and the local index on first run). Once indexed, agents query the codebase
      by intent -- "validate credentials", "error handling" -- instead of exact strings.</p>
      <p><strong>Result:</strong> fast, meaning-based lookups that feed precise context into the next step.</p>
    </div>
  </li>
  <li class="step step-primary">
    <div>
      <strong>Delegate the implementation</strong> <Badge variant="primary" text="developer agent" />
      <p>Hand the spec to the <code>developer</code> agent to write the code:</p>
      <pre><code>Implement the spec in SPEC.md</code></pre>
      <p><strong>What happens:</strong> the <code>developer</code> agent reads the specification,
      uses grepai to locate the relevant code, then implements, builds, and verifies the change.</p>
      <p><strong>Result:</strong> the feature is implemented against the spec, with semantic search keeping the agent grounded in your actual codebase.</p>
    </div>
  </li>
</Steps>

## What's next

<Callout type="tip" title="After the work is done">
  Here is what you can do next with the results:

  **[Prompt optimization](/brewtools/skills/text-optimize/)** -- reduce tokens in prompts and agents:
  ```bash
  /brewtools:text-optimize
  ```

  **[Remove AI artifacts](/brewtools/skills/text-human/)** -- humanize generated code and docs:
  ```bash
  /brewtools:text-human
  ```

  **[Scan for secrets](/brewtools/skills/secrets-scan/)** -- check changes for leaked credentials:
  ```bash
  /brewtools:secrets-scan
  ```
</Callout>
