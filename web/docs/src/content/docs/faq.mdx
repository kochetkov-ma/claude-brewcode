# FAQ

Click any question to expand the answer. Use the table of contents on the right to jump straight to a specific question.

<Spoiler id="update-plugins" title="How do I update brewcode plugins?">

<Callout type="tip" title="Recommended: use the skill">
  Run the [`/brewtools:plugin-update`](/brewtools/skills/plugin-update/) skill inside any Claude Code session. It runs the full update chain automatically, reports versions, and prompts you to reload.
</Callout>

If the skill is unavailable (very old install, plugin not yet installed, or Claude refuses to invoke it), paste the prompt below directly into the chat — Claude Code will execute every command for you:

<InstallPrompt plugin="all" mode="update" />

After all commands succeed, run `/reload-plugins` inside your Claude Code session.
If `/reload-plugins` is unavailable, type `exit` and run `claude` again.

</Spoiler>

<Spoiler id="install-too-old" title="My install is too old — the skill isn't available">

If [`/brewtools:plugin-update`](/brewtools/skills/plugin-update/) is missing (very old install or partial install), use the **Claude Code prompt** below. It tells Claude to run the marketplace update + every plugin update for you, no skill required:

<InstallPrompt plugin="all" mode="update" />

After Claude finishes, the [`/brewtools:plugin-update`](/brewtools/skills/plugin-update/) skill will be available for all future updates.

</Spoiler>

<Spoiler id="claude-refuses" title="Claude refuses to run the update commands">

Paste the following prompt directly into the chat. The coercive phrasing is intentional — it prevents Claude from summarizing or skipping steps.

<InstallPrompt plugin="all" mode="update" />

<Callout type="note" title="Easier alternative">
  Once any plugin update has succeeded, switch to the [`/brewtools:plugin-update`](/brewtools/skills/plugin-update/) skill — it handles refusal scenarios internally.
</Callout>

</Spoiler>

<Spoiler id="which-plugins" title="Which plugins are in the suite?">

| Plugin | Purpose | Install command |
|--------|---------|----------------|
| **brewcode** | Infinite task execution, 13 skills, 12 agents, 8 hooks | `claude plugin install brewcode@claude-brewcode` |
| **brewdoc** | Documentation tools: sync, memory, PDF, publishing | `claude plugin install brewdoc@claude-brewcode` |
| **brewtools** | Text utilities, [SSH](/brewtools/skills/ssh/), [deploy](/brewtools/skills/deploy/), [debate](/brewtools/skills/debate/), [plugin-update](/brewtools/skills/plugin-update/) | `claude plugin install brewtools@claude-brewcode` |
| **brewui** | AI image generation, design-to-code | `claude plugin install brewui@claude-brewcode` |

All four plugins install from the same `claude-brewcode` marketplace on GitHub.

</Spoiler>

<Spoiler id="restart-after-update" title="Do I need to restart after updating?">

Yes. Plugin changes only take effect after a reload.

<Steps>
  <li class="step step-primary">
    <div>
      <strong>Preferred: reload in session</strong>
      <p>Run <code>/reload-plugins</code> inside your active Claude Code session. This applies updates without closing the terminal.</p>
    </div>
  </li>
  <li class="step step-secondary">
    <div>
      <strong>Fallback: restart the CLI</strong>
      <p>Type <code>exit</code> to close the session, then run <code>claude</code> to start a new one. All updates will be picked up automatically.</p>
    </div>
  </li>
</Steps>

</Spoiler>

<Spoiler id="where-cached" title="Where are plugins cached?">

Plugins are stored locally after installation:

```
~/.claude/plugins/cache/<marketplace>/<plugin>/<version>/
```

For example:

```
~/.claude/plugins/cache/claude-brewcode/brewcode/3.4.51/
~/.claude/plugins/cache/claude-brewcode/brewtools/3.4.51/
```

The marketplace metadata lives at:

```
~/.claude/plugins/marketplaces/
```

</Spoiler>

<Spoiler id="check-version" title="How do I check my installed version?">

Run [`/brewtools:plugin-update`](/brewtools/skills/plugin-update/) inside a Claude Code session. It reports the installed and latest available version for each plugin and prompts you to update if a newer version exists.

<Callout type="note" title="Note">
  The <code>claude plugin list</code> CLI command does not exist. Version checking must be done through the <a href="/brewtools/skills/plugin-update/"><code>/brewtools:plugin-update</code></a> skill or by inspecting the cache directory above.
</Callout>

</Spoiler>
