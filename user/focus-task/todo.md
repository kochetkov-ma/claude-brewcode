### 03.02.2026
- ✅ v2.5.0 — Создание протокола `auto-sync` для документации + одноименный Агент который вызывается скиллом
  В документации описывается специальные раздел `auto-sync` - для каждого документа свой Алгоритм обновления - например кому-то нужно ходить в Интернет, кому-то нет. Если дефолтный протокол если явно не указан свой. 
  Документ тегируется `auto-sync-support` в шапке
  Есть скилл `auto-sync` принимает путь до папки / файлов или сканируется весь проект если ничего не задано или режим `auto-sync global` сканирует глобальные документы.
  `auto-sync` индексирует и сохраняет все `auto-sync-support` документы в проекте и глобальном файле и обновляет индекс (он сам тоже `auto-sync-support`)
  `auto-sync-support` подходит для ЛЮБОГО документа: скилл / агент / просто md / rules. Наверное для CLAUDE.md не стоит использовать `auto-sync-support`
  В `auto-sync` обычно описывается процедура КОРОТКО и ЧЕТКО по правилам LLM смотри скилл `text-optimize`. Обычно это стадия деления объема информации на блоки и параллельное исследование через SubAgents Explorer и далее через `auto-sync` Агента происходит Агрегация и обновление и фиксация версии и даты.
  Flow: `auto-sync` Скилл запускается в сессии, проверяет INDEX, читает переданные папки или файлы (дополнительно к индексу в зависимости от РЕЖИМА работы) на КАЖДЫЙ документ параллельно запускает `auto-sync` Агента, 
  который находит протокол в документе или использует дефолтный, бьет инфу на блоки, запускает Агентов на анализ параллельно, агрегирует, обновляет документ и фиксирует версию дата время. 
  И Финал - обновление Индекса если были найдены новые файлы, например когда в скилл была передана папака.
  Все Meta блоки `auto-sync` и версия-дата должны быть МАКСИМАЛЬНО сжаты так как это НЕ основной Контент документа.
  Если для конкретного документа не указан ОСОБЕННЫЙ протокол обновления то нужно использовать стандартный
  Протокол обновления по умолчанию: прочитать документ запомнить как контекст, найти все ссылки, разделить на три части и запустить параллельно Агнетов на исслдование кода или интернета 
  2 режима: global / project (default) для глобального работаем с INDEX.md в глобальной папке Клода, для проектного по умолчанию работаем с проектом и проектным INDEX.md
